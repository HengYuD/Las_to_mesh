# 室内 LAS 点云到轻量 OBJ 的工程化方案（面向射线追踪仿真）

## 1. 目标与约束

你当前的输入是 OmniSLAM R8 采集后、单层约 1.8GB 的 LAS 点云。目标输出并不是“高保真视觉模型”，而是“**结构正确 + 面数可控 + 法线稳定**”的仿真网格。

建议将验收指标拆为四类：

- **几何误差**：平面拟合残差（墙/地/顶）
- **拓扑质量**：非流形边、孔洞数量、法线一致性
- **轻量化程度**：OBJ 文件大小、三角面数量
- **仿真可用性**：是否保留关键结构（围护结构、主要隔墙、大开口）

## 2. 推荐系统架构

```text
LAS输入
  -> 数据标准化（坐标/单位/强度）
  -> ROI交互裁剪
  -> 噪声与游离点过滤
  -> 结构面分割（墙/地/顶）
  -> 结构约束平整（打薄）
  -> 网格化重建
  -> 简化与拓扑修复
  -> OBJ导出 + 质量报告
```

建议拆成 3 个可独立运行的服务：

1. **Preprocess Service**：大点云分块、索引、初筛。
2. **Geometry Service**：平面提取、重建、简化（核心算法层）。
3. **QA Service**：质量统计、可视化回看、版本对比。

## 3. 关键算法建议

### 3.1 区域外点云剔除（可视化框选）

- MVP：轴对齐包围盒（AABB）框选。
- 增强：支持多盒并集 + 排除盒（include/exclude）。
- 生产建议：保留原始点索引，便于回放和撤销。

### 3.2 游离点过滤

组合策略（按顺序）：

1. 体素下采样（5cm 起步，按仿真尺度调）
2. 统计离群滤波（SOR）
3. 半径离群滤波（ROR）
4. （可选）按强度/回波次数做异常剔除

### 3.3 墙/地/顶平整打薄

- 用 RANSAC 提取主平面簇，并以法向角度先验分类：
  - 地/顶：法向接近 Z 轴
  - 墙：法向接近 XY 平面
- 对每个平面簇进行投影（point-to-plane projection）实现“打薄”。
- 相邻墙体可施加正交约束（Manhattan World），降低锯齿和扭斜。

### 3.4 网格重建与轻量化

- 对结构面优先采用“轮廓线 + 三角化”而不是全局 Poisson。
- 非结构件可选局部 Poisson / BPA（可关闭）。
- 最后执行 QEM decimation，目标面数按仿真性能预算设上限（例如 50k~300k）。

## 4. 参数建议（第一版）

- 体素尺寸：`0.03 ~ 0.08 m`
- RANSAC 平面距离阈值：`0.01 ~ 0.03 m`
- 最小平面点数：依据场景大小动态设定（例如 20k）
- 墙体法向角阈值：`<= 10°`
- 地/顶法向角阈值：`<= 8°`

> 建议先在“一个楼层中的典型区域”调参，固化成配置模板后批处理。

## 5. 工程实现建议

- 处理 1.8GB LAS 时启用 **分块处理 + 中间缓存（npz/parquet）**。
- 每阶段输出中间结果（便于断点恢复）：
  - `roi.las`、`filtered.las`、`planes.json`、`mesh_raw.obj`、`mesh_slim.obj`
- 产线中记录版本信息：算法版本、参数 hash、输入文件 hash。

## 6. 你可以直接落地的“分阶段计划”

1. **Phase A（2~3周）**：打通自动化管线（无 GUI）。
2. **Phase B（2周）**：加入可视化框选、平面约束增强。
3. **Phase C（2周）**：仿真指标闭环（面数/误差/封闭性自动验收）。
4. **Phase D（持续）**：适配更多场景（楼梯间、大开间、曲面结构）。

## 7. 当前仓库脚手架对应关系

- `src/las_to_mesh/stages.py`：对应“阶段化处理”接口，便于替换为生产算法。
- `src/las_to_mesh/pipeline.py`：串联完整流程。
- `configs/pipeline.example.yaml`：参数化执行模板。
- `scripts/run_pipeline.py`：CLI 启动入口。
